# Continuidad para Codex

## Propósito de este ZIP
Este paquete está preparado para que Codex pueda:
- abrir el proyecto completo,
- entender su estado actual,
- leer el contexto de arquitectura y producto,
- y continuar el desarrollo sin perder el hilo.

## Qué debe leer primero
1. `docs/ESTADO_ACTUAL_PROYECTO.md`
2. `docs/CAMBIOS_POR_REALIZAR.md`
3. `docs/CONTINUIDAD_CODIGO_Y_ARQUITECTURA.md`
4. `docs/PROMPT_MAESTRO_PROYECTO.md`

## Qué debe preservar
- compatibilidad con Android Studio y VS Code
- estructura modular del simulador
- separación entre misión declarativa y runtime
- integración entre sistemas, UI, estado y tests
- orientación a simulación laboral real

## Qué no debe hacer
- reescribir el proyecto como una app distinta
- simplificar el flujo hacia una demo lineal
- perder trazabilidad entre misión, feedback, agenda y carrera
- agregar sistemas nuevos sin conectarlos a UI, estado y tests

## Estado funcional resumido
El proyecto ya incluye:
- 3 misiones jugables
- múltiples stakeholders
- sistema de tiempo y deadlines
- interrupciones y check-ins
- reuniones, decisiones y follow-ups
- mini-entregables y business review
- reputación por persona y por área
- trayectoria profesional emergente

## Próximo foco sugerido
El siguiente bloque más coherente es:
- choques de agenda y saturación laboral real

Luego:
- validación más profunda
- expansión de misiones
- organización más viva
- carrera más rica
